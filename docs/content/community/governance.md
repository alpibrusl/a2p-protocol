# Governance

How the a2p protocol is governed.

---

## Structure

```
Steering Committee
├── Core Protocol WG
├── Security WG
├── Implementation WG
├── Legal & Compliance WG
├── Registry WG
└── Community WG
```

---

## Decision Making

| Type | Process |
|------|---------|
| Editorial | Maintainer approval |
| Minor | 2 maintainer approvals |
| Significant | RFC + WG consensus |
| Breaking | RFC + SC approval |

---

## RFC Process

1. **Draft** — Author develops proposal
2. **Proposed** — WG review (2-4 weeks)
3. **Final Comment** — Community feedback (2 weeks)
4. **Accepted/Rejected** — WG decision
5. **Published** — Merged into spec
